# Support Policy

Onion Core is maintained as an open source library. Support is provided through
GitHub Issues and project documentation.

## Where to Ask

- Bug reports: open a GitHub Issue with a minimal reproduction.
- Feature requests: open a GitHub Issue and describe the use case.
- Security issues: follow `SECURITY.md`; do not open a public issue.
- General usage questions: use GitHub Issues when the answer would help future
  users.

## What to Include

For bugs, include:

- Onion Core version
- Python version
- operating system
- provider and model, if relevant
- minimal code sample
- traceback or structured error output
- whether the issue reproduces without external network calls

## Maintenance Expectations

The latest minor release receives priority fixes. Older minor releases are best
effort unless a maintainer explicitly marks them as supported.

The project does not provide private SLA-backed support unless that is arranged
outside the public repository.
